---
name: Commit after each feature
description: User wants git commits after every topic/feature change, not batched at the end
type: feedback
---

Always commit changes after completing each distinct topic or feature change.

**Why:** User wants granular commit history so changes are traceable per feature, not lumped together.

**How to apply:** After finishing a cohesive set of changes (e.g., "update proficiencies", "fix background gradients"), commit immediately before moving on to the next task.
